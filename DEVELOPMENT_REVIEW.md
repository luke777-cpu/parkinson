# 약효일지 전체 점검 및 수정 보고

작성일: 2026-07-22  
대상 버전: 테스트판 0.9.7

## 1. 결론

현재 앱에는 이미 다음 기능이 구현되어 있다.

- 복약·상태·출력·증상·생활기록 저장
- 하루 요약, 추이, 진료용 보고서
- `computeCurve()` 기반 상대적 추정 약효곡선
- `renderSimChart()` 기반 현재안·가상안 비교
- 복약 뒤 출력 평균과 생활요인 연동

이번 수정에서는 기존 기능과 저장 키 `yakhyo_log_v1`을 유지하면서 명확한 오류와 배포 문제를 고쳤다. 새 증상–곡선 분석 기능은 운영용 `index.html`에 연결하지 않고 `engines/`, `tests/`, `demo/`에 독립적으로 구현했다.

## 2. 기존 데이터와 계산 위치

### 운영 데이터

- 저장소: 브라우저 `localStorage`
- 저장 키: `yakhyo_log_v1`
- 복약 이벤트: `{type:"med", drug, dose, ts, memo}`
- 상태 이벤트: `{type:"state", state, ts, output, gait, symptoms, dysk, riseResult}`
- 생활 이벤트: `{type:"life", kind, ts, ...}`
- 약 목록: `db.meds`
- 설정: `db.settings`

### 기존 계산 함수

- 약물 프로파일: `MED_KINETICS`, `kineticFor()`
- 1회 복약 상대 곡선: `doseCurve()`
- 하루 추정 곡선: `computeCurve()`
- 곡선 표시: `svgCurve()`
- 가상안 비교: `simDoseCurvePts()`, `renderSimChart()`
- 참고 LEDD: `leddFactorFor()`, `simTotalLEDD()`
- 복약 후 출력 평균: `medOutputCorrelation()`

기존 추정 곡선은 복약 시간·용량·단순화된 약물 프로파일로 계산된다. 실제 출력과 보행 기록은 곡선 계산에 섞이지 않고 별도 표시하는 것이 코드의 실제 동작이다.

## 3. 이번에 고친 문제

1. **PWA 아이콘 404**  
   `manifest.json`과 `sw.js`가 존재하지 않는 `icons/` 폴더를 가리키던 문제를 실제 루트 파일 경로로 수정했다.

2. **오래된 화면이 계속 남는 문제**  
   서비스워커 캐시를 `yakhyo-v0.9.7`로 올리고, 화면 이동은 network-first, 정적 파일은 cache-first 갱신 방식으로 분리했다.

3. **출력선 10배 오류**  
   출력은 이미 0–100인데 곡선과 시뮬레이터에서 다시 10배 하던 계산을 제거했다. 보행 점수만 0–10이므로 10배 표시를 유지했다.

4. **기록 공백을 실제 출력처럼 이어 그리던 문제**  
   가장 가까운 상태를 하루 전체에 반복 적용하던 방식을 제거했다. 실제 입력점만 표시하고 4시간이 넘는 공백은 선으로 연결하지 않는다.

5. **곡선 설명과 실제 코드의 불일치**  
   출력·보행·증상이 추정곡선 계산에 포함된다는 문구를 삭제하고, 같은 시간축에 별도로 표시된다고 바로잡았다.

6. **백업 불러오기 안전성**  
   파일 크기 제한, 이벤트 종류·시간·숫자 범위·약 예약 시각·문자열 길이 검증을 추가했다. 불러온 HTML성 문자열이 화면에 실행되지 않도록 데이터 구조를 정규화한다.

7. **개인정보 안내 부족**  
   브라우저 음성인식은 환경에 따라 제공자 서버에서 처리될 수 있음을 명시했다. 앱은 음성 파일을 저장하지 않으며 음성 권한 없이도 수동 기록을 사용할 수 있다.

8. **버전 불일치**  
   0.9.2와 0.9.6으로 섞여 있던 화면·번역·캐시 버전을 0.9.7로 통일했다.

## 4. 독립 분석 엔진

- `engines/drugCurveEngine.js`: 기존 상대 곡선 함수를 재현하고 전날 잔여효과를 포함한다.
- `engines/outputCurveEngine.js`: 직접 출력 > 도움받은 출력 > 상태 기본값 순서로 처리하며 4시간 이상 공백을 추정하지 않는다.
- `engines/symptomCurveEngine.js`: 증상 시점의 곡선 높이·기울기·직전 복약시간·실제 출력을 계산한다.
- `engines/curveMismatchEngine.js`: 추정 높음+출력 낮음, 추정 낮음+출력 높음의 반복을 3회 이상부터 후보로 표시한다.
- `engines/appDataAdapter.js`: 현재 앱의 이벤트를 바꾸지 않고 표준 분석 자료로 변환한다.
- `demo/engine-demo.html`: 가상 기록으로 곡선·출력·증상과 근거표를 보여준다.

모든 결과에는 표본수, 분포, 신뢰 수준, 원자료와 의료 안전 문구가 포함된다. “약이 과하다·부족하다”, “용량을 늘리거나 줄이라”는 결론은 생성하지 않는다.

- 추정곡선–실제 출력 불일치

- 출력 0–100 보존 및 4시간 공백 차단

- 곡선 구간 분류 유효성

## 6. 남은 위험과 다음 승인 항목

1. `assetlinks.json`의 Play App Signing SHA-256 지문은 실제 값이 없어 자리표시자로 남아 있다. 이 값이 없으면 Android 앱과 웹주소 연결 검증이 통과하지 않는다.
2. 기존 `MED_KINETICS`와 `leddFactorFor()`는 단순화된 참고 계산이다. 공식 LEDD 또는 임상 예측으로 공개하기 전 약물별 파라미터와 환산식을 별도 임상 검토해야 한다.
3. 독립 분석 엔진은 아직 운영 화면에 연결하지 않았다. 실제 자료 복사본으로 검증하고 승인한 뒤 연결하는 것이 안전하다.
4. 실제 Chromium 실행파일이 없어 설치 배너와 서비스워커 UI까지 자동 시험하지는 못했다. 대신 DOM 실행환경에서 상태·복약 입력, 곡선·시뮬레이터 렌더링, 출력 좌표, 악성 백업값 정규화를 실행했고 모두 통과했다. 모든 URL·JSON·JavaScript·아이콘 크기와 캐시 목록도 별도로 검증했다.
